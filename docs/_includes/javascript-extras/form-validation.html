Form validation validator.js

Form validation ensures users provide necessary data and that the data is propertly entered. This client-side jQuery validation plugin, named Bootstrap Validator is developed by Nguyen Huu Phuoc and maintained on this GitHub repo. The Bootstrap Validator site also includes documentation and examples.

Client-side vs. server-side validation

While client-side validation is good for immediate user feedback on the data that is being entered, if you are validating input due to security concerns, it is recommended to have server-side validation also applied to the input since it is relatively easy to disable Javascript validation. Read this Stack Overflow topic for a broader discussion on this topic.

Included validation types

Boostrap Validator can validate almost anything. However, for the base build and in the interest of keeping the file size to a minimum, the Arrow Framework's version includes only the validations listed below.

We have also listed the validations that are available but not added into this version of the plugin. If you need too add any back, these validators can be found either by downloading the current full plugin, or by pulling the standalone validators from GitHub and adding them to your build.

To validate this form, you would need the following Javascript:

{% highlight js %} $('.registerForm').bootstrapValidator({ message: 'This value is not valid', feedbackIcons: { valid: 'fa fa-check', invalid: 'fa fa-times', validating: 'fa fa-refresh' }, fields: { username: { message: 'The username is not valid', validators: { notEmpty: { message: 'The username is required and cannot be empty' }, stringLength: { min: 6, max: 30, message: 'The username must be more than 6 and less than 30 characters long' }, regexp: { regexp: /^[a-zA-Z0-9_]+$/, message: 'The username can only consist of alphabetical, number and underscore' } } }, email: { validators: { notEmpty: { message: 'The email is required and cannot be empty' }, emailAddress: { message: 'The input is not a valid email address' } } } } }); {% endhighlight %}

Usage

Via data attributes

Plugin options can be triggered by the data-bv-* tag. However, unlike Bootstrap plugins, this plugin cannot be initiated via data-attributes. You will need to also instantiate the validation object via Javascript, as shown below. Full examples and documentation on how to use data-attributes for this plugin can be found here.

Via JavaScript

Call the bootstrapValidator object using jQuery. You may also add options to the object this way, or use data-attributes in the markup.

{% highlight js %} $(formSelector).bootstrapValidator({ excluded: ..., feedbackIcons: ..., live: 'enabled', message: 'This value is not valid', submitButtons: 'button[type="submit"]', trigger: null, fields: ... }); {% endhighlight %}

Options

A full list of options as well as more examples can be found at the Bootstrap Validator site.

Option Equivalent HTML attribute default
container data-bv-container n/a
excluded data-bv-excluded [':disabled', ':hidden', ':not(:visible)']
feedbackIcons data-bv-feedbackicons-valid, data-bv-feedbackicons-invalid, data-bv-feedbackicons-validating { valid: null, invalid: null, validating: null }
group data-bv-group '.form-group'
live data-bv-live 'enabled'
message data-bv-message 'This value is not valid'

threshold data-bv-threshold null
