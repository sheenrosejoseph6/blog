What's included

The Arrow framework is downloadable in two forms, within which you'll find the following directories and files, logically grouping common resources and providing both compiled and minified variations.

jQuery required

Please note that all Arrow JavaScript plugins require jQuery to be included, as shown in the starter template.

Precompiled framework files

Once downloaded, unzip the compressed folder to see the structure of (the compiled) this framework. You'll see something like this:

{% highlight bash %} arrow-framework/ ├── css/ │ ├── arrow-base.css │ └── arrow-base.min.css │ ├── arrow-components.css │ └── arrow-components.min.css ├── fonts/ │ ├── arrow/ │ └── font-awesome/ ├── images/ │ └── webicons/ └── js/ ├── arrow-base.js ├── arrow-base.min.js ├── arrow-components.js └── arrow-components.min.js {% endhighlight %}

This is the most basic form of this framework: compiled files for quick drop-in usage in nearly any web project. We provide compiled CSS and JS, as well as compiled and minified CSS and JS. Fonts from Font Awesome as well as Arrow's base display fonts are included and images from Fairfield Creative's FC Webicons.

Source code

The Arrow framework source code download includes the precompiled CSS, JavaScript, font and image assets, along with source SASS, JavaScript, and documentation. More specifically, it includes the following and more:

{% highlight bash %} arrow-framework/ ├── dist/ │ ├── css/ │ ├── fonts/ │ ├── images/ │ └── js/ └── docs/ │ └── examples/ ├── fonts/ ├── images/ ├── scss/ └── scripts/ {% endhighlight %}

The scss/, scripts/, fonts/ and images/ folder contain the source code for our CSS, JS, icon/display fonts and webicons (respectively). The dist/ folder includes everything listed in the precompiled download section above. The docs/ folder includes the source code for our documentation, and examples/ of this framework usage. Beyond that, any other included file provides support for packages, license information, and development.

SASS source code structure

If you download the source code, you'll need to know something about the structure of the SASS files within.

Unzip the compressed folder of the source code and look within the folder named scss to see this framework's pre-complied structure.

{% highlight bash %} scss/ ├── framework/ │ ├── *.scss ├── prop/ │ ├── _application.scss │ ├── _arrow-styles.scss │ ├── _fonts.scss │ └── _variables.scss ├── vendor/ │ ├── animate.css/ │ ├── bourbon/ │ └── font-awesome/ │ └── foundation/ │ └── neat/ │ └── webicons/ ├── arrow-base.scss └── arrow-components.scss {% endhighlight %}

The framework folder contains all the base style files including the CSS reset, grid, typography and UI objects that come from Bootstrap, Foundation and elsewhere.

The prop folder contains four files _application.scss, _arrow-styles.scss, _fonts.scss and _variables.scss. This folder is where the custom files for your site should go. It also includes the fonts and variables file that will determine the color palette, spacing and typography choices.

The _application.scss file serves as a complier for the custom SASS files you create — similar to the arrow-base.scss and arrow-components.scss files contained in the root folder. When you open arrow-components.scss, for example, you will notice that _application.scss is one of the last file within the compliation chain. arrow-base.scss and arrow-components.scss is also where you can comment out the styles you may not need from the master framework files.

Finally, the vendor folder should contain any third-party plugin style files you want to add into your project. By default, the animate.css, bourbon.scss, font-awesome.scss, foundaton, neat and webicon files are contained here.

SASS compilation

If you download the original files, you need to compile this framework's SASS files into usable CSS. To do that, you will need a complier. Here are some options:

Third party compliers

Compliers run alongside your browser and recompile your SASS files per save. For Mac users, CodeKit or LiveReload are good options. For PC users, Prepos or Koala are reccommended.

Command-line compliers

SASS is available as a Ruby Gem which can be run from the command line. If you already have Ruby Gems installed on a Linux machine, installing SASS is easy with the following command:

{% highlight bash %} gem install sass {% endhighlight %} Ruby comes pre-installed on a Mac. For Windows machines, you may need to first install Ruby. Ruby Installer is the quickest way to accomplish that. More documentation on how to work with SASS from the command line is available at the SASS website.
